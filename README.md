## Overview
 
### The Challenge
 
Users should be able to:
 
- Complete the form and see a success toast message upon successful submission
- Receive form validation messages if:
  - A required field has been missed
  - The email address is not formatted correctly
- View the optimal layout depending on their device's screen size
- See hover and focus states for all interactive elements on the page

## My Process
 
### Built With
 
- Semantic HTML5 markup
- CSS custom properties
- Flexbox
- Vanilla JavaScript for form validation
- Mobile-first workflow
### What I Learned
 
Working on this project helped me strengthen my understanding of form design and validation. A few highlights:
 
- Structuring accessible forms using proper `<label>` and `for` attribute pairing
- Styling radio buttons and checkboxes with custom CSS while keeping them functional
- Implementing client-side validation logic in JavaScript to show and hide error messages
- Managing hover, focus, and active states using CSS pseudo-classes
- Building a responsive two-column layout using Flexbox that collapses gracefully on mobile
 
### Continued Development
 
In future projects I'd like to:
 
- Learn and use `aria-describedby` and `aria-invalid` attributes for better screen reader accessibility
- Add keyboard-only navigation support so the form is fully accessible
- Explore smoother toast/success notification animations
- Look into form validation libraries for more scalable projects
